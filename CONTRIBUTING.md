# Contributing to Aglyphs

Thank you for considering contributing to Aglyphs!

## Philosophy

Aglyphs is intentionally **minimal, precise, and text-native**. New glyphs or operators should only be added when they solve a clear, recurring need in agentic systems that cannot be expressed cleanly with existing symbols.

## How to Contribute

### 1. New Patterns (Most Welcome)

The easiest and highest-impact contribution is adding a new ready-to-use pattern in `AGLYPHS-NOTATION.md`.

**Requirements:**
- Clear, descriptive name
- One-line Aglyph formula
- Brief explanation of when to use it
- Must be useful for Grok, Claude, or general LLM agent workflows

### 2. New Glyphs / Operators

Only propose new symbols when existing ones are insufficient.

When adding a glyph:
- Add it to the Core Primitives or Flow Operators table
- Provide strong justification and real usage examples
- Ensure it renders cleanly in monospace fonts

### 3. Renderers & Tooling

- CSS for better glyph display
- JavaScript playground
- Mermaid / PlantUML plugin
- VS Code / JetBrains syntax highlighting

### 4. Documentation & Examples

- Real-world examples from production agent systems
- Integration guides (Notion, Obsidian, Linear, GitHub issues, etc.)

## Development Process

1. Fork the repository
2. Make your changes
3. Update `AGLYPHS-NOTATION.md` if adding glyphs or patterns
4. Test rendering in multiple Markdown viewers
5. Submit a Pull Request with clear description
